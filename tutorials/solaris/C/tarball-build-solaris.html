Who This Is Information For

This information is for developers who want to download tarballs from the GNOME website, then build the tarballs on systems running the Solaris™ 8 operating environment, and the Solaris 9 operating environment.

Before You Download and Build a Tarball

Before you download and build a tarball of a GNOME application, you need to perform the following steps:

Check that the development packages are installed on your system

You need to ensure that the development packages are installed in the /opt/sfw area. Perform the following command to display information about the installation status of the development package:

% pkginfo SUNWgnome-base-libs-devel

Set up the build environment

Use the following packages contained on the companion CD delivered with the CD for the Solaris operating environment:

  • m4-1.4

  • autoconf-2.53

  • automake-1.6.1

  • gcmn-1.0

  • gmake-3.79.1

  • libtool-1.4.2

You can also download the build environment packages from the freeware website hosted by Sun Microsystems. You must register on the freeware website before you can download the packages.

Please note the listed order of the packages, as some of the packages have dependencies. Install the packages as instructed on the freeware website.

Setup environmental variables

You should set the following environmental variables:

  • DE_HOME /opt/sfw

  • PATH /opt/sfw/bin:/usr/sfw/bin:$PATH

  • LD_LIBRARY_PATH /opt/sfw/lib:/usr/sfw/lib:$LD_LIBRARY_PATH

  • CC cc

  • CPPFLAGS -I/usr/include -I/usr/sfw/include

  • CFLAGS -xO4 -xspace -xarch=v8a -xstrconst -Wl,-zignore -I/usr/include

  • LDFLAGS -i -xO4 -xspace -xarch=v8a -L/usr/sfw/lib -R/usr/sfw/lib

  • PKG_CONFIG_PATH /usr/lib/pkgconfig

Verify the Correct Version of the Forte Compiler

Forte 6 update 2, or FD 7 (C 5.4) are the required versions of Forte to ensure GNOME builds.

Downloading the Tarball and Building the Application

When you have completed all of the steps in the section called “Before You Download and Build a Tarball”, then perform the following steps to download the tarball and build the application:

Download the tarball

Download and unpack a source tarball from the GNOME website , using the Get GNOME link.

Configure and compile the application

Run the following commands:

  1. %./configure --prefix=/usr

  2. %make

Check dependencies

Some binaries might have dependencies on other binaries that are not part of the GNOME Desktop or of the GNOME developer pack. In these cases, you need to download and build the dependency files. Some tarballs contain a DEPENDS file that lists the dependencies. If there is DEPENDS file, the script informs you that a specific binary or library is missing.
